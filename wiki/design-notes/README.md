# Design Notes

## 목적

이 폴더는 기존 게임 분석 결과를 사용자의 게임 개발 의사결정으로 변환한 2차 Wiki 문서를 보관한다.

## 예시 질문

- 특정 게임의 핵심 재미를 내 게임에 적용하려면 어떤 시스템이 필요한가?
- 두 게임의 차별점을 조합하면 어떤 디자인 방향이 나오는가?
- 특정 장르 태그를 가진 게임들의 공통 패턴은 무엇인가?

## 관리 규칙

- 단순 비교는 `wiki/comparisons/`에 저장한다.
- 개발 적용 메모, 시스템 아이디어, 디자인 가설은 `wiki/design-notes/`에 저장한다.
- 근거가 된 게임 문서와 비교 문서를 반드시 연결한다.
