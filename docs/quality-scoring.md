# Quality Scoring

이 문서는 Game Analysis LLM Wiki가 게임 분석 페이지의 품질을 어떻게 평가하는지 설명한다.

## 목적

품질 점수는 분석이 “그럴듯해 보이는가”가 아니라, 실제 운영 가능한 위키 문서로서 얼마나 검증 가능한지를 판단하기 위한 지표다.

점수는 다음 목적에 사용된다.

- 보강이 필요한 게임 문서 우선순위 결정
- 공식/공개/유저 경험 출처의 균형 확인
- Core Loop 분석이 출처 기반인지 추론 기반인지 구분
- 과제 평가자가 파이프라인의 검증 방식을 이해할 수 있도록 설명

## 점수 산식

총점은 0-100점이다.

| 항목 | 점수 |
|---|---:|
| 공식/스토어 계열 출처 존재 | +25 |
| 공개 참고 출처 존재 | +15 |
| 플레이 경험/유저 반응 신호 존재 | +15 |
| Core Loop 단계 수와 깊이 | 최대 +15 |
| 주요 시스템/mechanics 수 | 최대 +10 |
| schema.validate 통과 | +15 |
| 검증 필요 사항 | 최대 -15 |

## 등급

| 등급 | 점수 범위 | 의미 |
|---|---:|---|
| strong | 80-100 | 출처, 구조, 플레이 경험 신호가 충분하다. |
| usable | 60-79 | 실사용 가능하지만 일부 보강 여지가 있다. |
| weak | 35-59 | 분석 초안으로는 의미가 있으나 근거가 약하다. |
| seed | 0-34 | 조사/근거 보강이 우선 필요한 상태다. |

## Core Loop Confidence

Core Loop 항목에는 다음 confidence가 붙는다.

| confidence | 의미 |
|---|---|
| high | 출처 id와 직접 연결되는 분석이다. |
| medium | 태그, 장르, 요약 정보에 근거한 간접 분석이다. |
| low | 출처보다는 구조적 추론에 가까운 분석이다. |

예시:

```md
1. 환경 탐색 [S1][confidence: high]
2. 빌드 조정 [inference] [confidence: low]
```

## 운영 규칙

- 점수가 낮은 문서를 삭제하지 않는다.
- 낮은 점수는 실패가 아니라 보강 우선순위다.
- 새 검색 없이 기존 문서만 보강할 때는 `Quality Backfill` 스킬을 사용한다.
- 새 출처와 유저 반응까지 보강할 때는 `Analyze New Game` 또는 `Refresh Evidence`를 사용한다.

## 현재 해석 기준

- `Balatro`, `Slay the Spire`처럼 점수가 낮은 문서는 공식/공개/유저 경험 출처가 부족한 레거시 문서일 가능성이 높다.
- `Hades`, `Subnautica 2`처럼 strong인 문서는 출처와 Core Loop 분석이 비교적 안정적이다.
- `MapleStory`처럼 usable인 문서는 기본 분석은 가능하지만 유저 경험 신호나 검증 필요 사항 보강이 남아 있다.
